The Swift Migrations demo site is an example corporate website. It uses Perch Core along with our free add ons Blog and Forms. Swift was designed by Laura Kalbag.

With the Swift Migrations website we have tried to show the process of adding Perch to a site already built using HTML and CSS. It demonstrates how Perch does not require the use of any themes, you can build your site in the usual way, using your favourite tools, and just add Perch at the end to make the site content editable.

The entire process of building the Swift site is documented in our video tutorial, where you can even download the starting point un-Perched code. As with the other demos, the completed page and template code can be found on GitHub.

In our walkthrough of the default site we pointed out some of the basic content management capability of Perch, most of which is relevant in the Swift site too. However here are some other things to look at.

Shared Regions

There is often content on a site that needs repeating in various places. If it has to be re-entered each time that is time-consuming and also prone to mistakes, so Perch gives you away to make certain content shared. On the Swift site our contact details are created as a Shared Region. If you edit the contact details in the Perch Admin under Shared you will see that they change in the footer of the site on each page and also on the contact page itself.

This video demonstrates how we create the Shared Region.

Repeating Content with Multiple Item Regions

When setting up a Region in Perch, you can optionally set that Region to allow multiple items. You can see this in action if you take a look at the Team page on the Swift Migrations site.

In Region options you also have a choice of Editing “All on one page” or using List Detail. If you look at the Team page in the Perch Admin, it is set up to use List Detail, click Region Options and check Edit all on one page and you can see that alternate view. You are likely to find both options useful. If a Multiple Item Region is just allowing the editor to add list items then all on one page makes sense, however larger and more complex templates are probably more suited to List Detail view.

Re-purposing content with perch_content_custom()

On the homepage of our site are some Testimonals that display at random, these are pulled from the Testimonials page of the site and use perch_content_custom() to grab two testimonials from that region sorted randomly. Take a look at the video to see how this works and don’t forget that the page code and templates can be found on GitHub if you want to implement something like this yourself.

Using Perch Add-ons

We are committed to a transparent price for Perch. When you buy a Perch license you also get access to all of our free apps and other add-ons for your site. We won’t charge you extra when you need to add a blog or create a contact form! The Swift site uses two of our add-ons, to see more in action check out the Nest Running Club Demo.

Perch Forms

The contact page has a form, this form is created using Perch Form template tags and, with the free Forms App installed, can be configured to both email the responses somewhere and also store them in a way that means they can be viewed in admin and downloaded as a csv to be opened in Excel or Numbers, or imported into another system.

Perch makes adding forms simple and we have fully implemented the HTML5 forms field types so if your user is using a supporting browser the HTML5 Validation will be used, if not the forms are backed up with server-side validation implementing those same types.

The Blog

In addition to the Forms App, the Swift site also has a blog using our free Blog App. This app offers a blog with archives, categories, tags and comments and as with any content in Perch you can choose to edit using Textile, Markdown or HTML with any of our available WYSIWYG plugins.

You can add your own custom fields to the Blog, using template tags in exactly the same way you create templates for Perch Content, so if you would like your posts illustrated with an image or to contain some other information to work with your design and requirements you can do that easily.
