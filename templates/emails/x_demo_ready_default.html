Editing content using Perch

The default demo site has been set up as a mini tutorial in how to edit content in Perch. Take a look at the homepage of the site, there is a welcome message there. In Perch admin, from the list of pages select Homepage and then in the list of homepage Regions select “Welcome message”, you can edit the text there.

Drafts and Preview

In some simple content editors saving an edit will push it live right away, which can be a bit scary for clients who would like to check it first.

When saving an edit you can check the Save as draft checkbox, you will then get a link to preview how things will look before publishing the change.

Another useful feature is Undo. Perch saves a history of content changes so if an editor does something and realizes it was a mistake they can just hit undo and the content will go back to the way it was before.

How do those content regions get there?

It is super simple to start editing basic content like this welcome text in Perch. You add Perch to the page, add a Perch Region tag with the name of the Region, refresh your page and that Region will show up in the Perch admin. There you can select a template. You can see exactly how this works in this video.

What are Perch Templates and can I make my own?

Perch ships with some basic templates, these are just fragments of HTML with Perch tags in them. As shown in the above video, you can start to edit basic content using our default templates. The real magic of Perch comes when you start to create your own as they allow you to create your own markup and style it just as you please, without worrying about what your client might do to it.

The Intro Block template used for the Welcome text looks like this:

<<h1><perch:content id="heading" type="text" label="Heading" title="true" size="xl" /></h1>
<perch:content id="text" type="textarea" label="Text" textile="true" editor="markitup" imagewidth="640" imageheight="480" />

So here we have a single line text field and a textarea field. You can find out more about template tags in the documentation and we have a number of videos that cover different aspects of working with templates in Perch.

By the way, all of the page and template code for this demo is available on GitHub so you can have a look. If any of the templates are useful in your project once you have a Perch license feel free to grab and use them either as is, or as a starting point to edit.

My clients won’t get on with Textile

We use the MarkItUp Editor with Textile on our default templates and you’ve seen it in action in this demo. We understand that many people would prefer to use some kind of WYSIWYG editor – the problem with these is that they do tend to introduce nasty markup. As a core concept in Perch is that you can protect your markup and design we don’t use one on our default templates.

That said, if you want a WYSIWYG we have them enabled as Plugins for Perch. There are a number of these available as free add-ons from our website and you can even create your own Plugin if you want to use an editor that we don’t have our own wrapper for.

On the demo, take a look at the Staff Profiles Region of the About Page. The Biography can be edited using the Redactor Editor. You would just need to download and install Redactor as described here and then use it on a textarea field like so:

<perch:content id="bio" type="textarea" label="Biography" editor="redactor" size="s" html="true" order="5" />

Template Conditionals

While we are looking at the Staff Profiles region we can see something interesting. In the template is the following section:

<perch:if exists="image">
<perch:content id="image" type="image" label="Photo" width="80" height="80" density="2" crop="true" order="6" output="tag" class="photo" />
<perch:else />
<img class="photo" src="https://www.gravatar.com/avatar/< perch:content id="email" type="email" label="Email address" required="true" order="4" hash="md5" />?s=160&amp;d=mm" width="80" height="80" />
</perch:if>

We are using Perch Conditionals here to check and see if an image has been uploaded. If it has we display the image, if not we see if the user has a Gravatar for their email address and display that. You can find out more about using Perch Conditionals in the documentation or in this video from the Video Tutorial.
